# Customer Feedback Taxonomy

Owner: Product
Last reviewed: 2026-02-14

## Purpose

This taxonomy helps teams tag feedback consistently across Support, Sales, Customer Success, and Product.

## Tags

- `bug`: expected behavior is broken;
- `usability`: user can complete workflow but has confusion or friction;
- `missing-feature`: workflow cannot be completed without new capability;
- `integration`: request involves external systems;
- `reporting`: request involves metrics, exports, dashboards, or audit views;
- `admin-control`: request involves permissions, settings, or governance;
- `performance`: workflow is slow or unreliable;
- `docs`: user needed clearer documentation.

## Required Context

Every feedback item should include user role, workflow, current workaround, customer impact, frequency, and source link.
