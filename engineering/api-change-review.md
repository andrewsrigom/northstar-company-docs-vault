# API Change Review

Owner: Engineering
Last reviewed: 2026-02-18

## Purpose

This review process reduces customer impact from API changes.

## Review Required

API review is required for new public endpoints, breaking changes, authentication changes, rate limit changes, webhook payload changes, deprecated fields, and changes to error semantics.

## Required Notes

Design notes should include use case, affected customers, request and response examples, compatibility plan, migration plan, observability, documentation owner, and support impact.

## Deprecation

Breaking changes require Product and Customer Success communication plans. Do not remove a public field or endpoint until migration criteria are met or leadership approves the exception.
