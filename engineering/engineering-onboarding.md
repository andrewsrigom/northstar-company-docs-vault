# Engineering Onboarding

Owner: Engineering
Last reviewed: 2026-03-20

## Purpose

This checklist helps new engineers become productive while following access, security, and release expectations.

## First Week

- complete company onboarding;
- enable MFA and password manager;
- request repository and issue tracker access;
- set up local development environment;
- read service ownership pages;
- shadow one code review;
- review release management and incident response docs.

## First Month

New engineers should ship a small change, participate in an on-call shadow if applicable, read one recent incident review, and meet Product, Support, and Customer Success partners.

## Access

Production access is not granted by default. It requires manager approval, system owner approval, security training, and a documented business need.
