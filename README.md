# Northstar Company Internal Knowledge Demo

This safe demo vault simulates a generic mid-size company using VaultMind as an internal knowledge assistant. It contains fictional policies, runbooks, FAQs, working notes, archived pages, and known documentation gaps across departments.

The demo is designed for sales and product validation:

- Employees can ask practical workplace questions without knowing where the answer lives.
- Department owners can manage their own source folders.
- Useful answers can be promoted into reviewed answers.
- Missing or weak answers can become documentation gaps.

## Departments

- Company: handbook entry points, operating cadence, glossary, decision logs, documentation standards.
- People & HR: timekeeping, PTO, benefits, onboarding, parental leave.
- IT & Security: access requests, MFA, device security, incident response.
- Operations: procurement, office requests, process handoffs, vendor onboarding.
- Finance: reimbursements, budget approvals, invoice intake.
- Legal & Compliance: vendor contracts, record retention, privacy review.
- Marketing: campaign requests, brand approvals, customer-facing copy.
- Support: internal escalation, response templates, policy triage, service ownership.
- Engineering: onboarding, releases, service ownership, incident reviews, API changes.
- Product: discovery intake, roadmap planning, beta programs, launch readiness, feedback taxonomy.
- Sales: lead routing, discount approvals, security questionnaires, demo environment rules.
- Customer Success: onboarding, QBR prep, renewal risk, customer escalation communication.
- Data Analytics: metric definitions, dashboard requests, data access, experiment readouts.
- Archive: stale, draft, or historical notes that simulate real enterprise knowledge sprawl.

## Demo Scenarios

The vault is intentionally shaped like a mixed Notion, Confluence, and shared-drive export. It includes:

- current policies with clear owners;
- draft guidance that should not be treated as final;
- archived pages that mention superseded rules;
- cross-functional handoffs where multiple teams approve different parts;
- repeated concepts with different wording to exercise retrieval quality;
- Markdown image references for future rendering and media support tests;
- known gaps that a knowledge assistant should identify instead of guessing.

## Example Questions

- What approval do I need for international remote work?
- Can I paste customer data into an AI tool?
- Who owns vendor renewal decisions?
- What happens if a customer asks for a security questionnaire?
- When does Product need Legal review for a beta?
- How should Support handle a policy question with no clear source?
- What is the difference between a business owner and a system owner?
- Which team approves dashboard access to customer data?

## Demo Safety

All names, tools, thresholds, and process details are fictional. Do not treat these documents as legal, HR, security, or financial advice.
