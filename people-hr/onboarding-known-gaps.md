# Onboarding Known Gaps

Owner: People Operations
Status: documentation gap
Last reviewed: 2026-06-01

## Purpose

This page captures known gaps in the Northstar employee onboarding documentation. It should help Support, managers, and People Operations avoid inventing answers when the handbook is incomplete.

## Missing 30/60/90 Plan

Northstar does not have a company-wide 30/60/90 onboarding plan. Some departments keep informal milestone notes, but there is no reviewed template that defines expected learning goals, manager check-ins, role milestones, or readiness signals across the first 90 days.

Managers should not present team-specific milestone notes as company policy. If a new hire asks for 30/60/90 expectations, Support should route the question to the manager and mark the case as a documentation gap.

## Contractor Onboarding Gap

The first-week onboarding guide covers employees only. It does not define contractor, temporary worker, or agency worker onboarding. Missing details include background checks, equipment ownership, system access duration, training requirements, and offboarding timing.

## Access Bundle Gap

The default access bundle is not listed in the People & HR onboarding guide. IT Security owns system access rules, but People Operations owns the onboarding checklist. Until the handoff is documented, managers must confirm role-specific tools before IT grants access.

## Suggested Fix

People Operations should create a reviewed onboarding template with:

- employee 30/60/90 milestones;
- contractor onboarding variant;
- manager checklist;
- IT access handoff;
- training requirements;
- review date and owner.
