# AI Tool Usage

Owner: IT Security and Legal
Status: draft
Last reviewed: 2026-04-12

## Purpose

This draft guidance explains how employees may use generative AI tools with company information. It is intentionally conservative until Legal and IT Security publish a final policy.

## Allowed Low-Risk Use

Employees may use approved AI tools for public information, generic writing help, summarizing non-sensitive internal notes, brainstorming, and code examples that do not include secrets or customer-specific data.

## Not Allowed Without Approval

Do not paste customer data, employee records, contracts, security findings, credentials, source code containing secrets, non-public financials, regulated data, or privileged legal material into unapproved AI tools.

## Review Required

Legal and IT Security review is required before using AI tools for customer-facing decisions, automated employee decisions, regulated workflows, vendor assessment, or processing Confidential or Restricted data.

## Documentation Gap

This draft does not yet define the approved tool list, retention settings, model training controls, or exception process. Employees should route uncertain cases to IT Security.
