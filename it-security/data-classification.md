# Data Classification

Owner: IT Security
Last reviewed: 2026-03-14

## Purpose

This page defines simple data handling categories for employees and system owners.

## Public

Public data is approved for external release. Examples include published marketing pages, public documentation, approved press materials, and public job postings.

## Internal

Internal data is intended for employees and approved contractors. Examples include internal process docs, team notes, non-sensitive project plans, and operating metrics that are not customer-specific.

## Confidential

Confidential data requires business need and approved storage. Examples include customer data, employee records, contracts, security findings, pricing exceptions, non-public financials, and product roadmap details.

## Restricted

Restricted data requires explicit approval and additional controls. Examples include secrets, private keys, payroll details, government IDs, health information, legal privilege material, and breach investigation records.

## Handling Rule

When unsure, treat data as Confidential and ask IT Security or Legal before sharing externally, uploading to a new vendor, or using it in an AI tool.
