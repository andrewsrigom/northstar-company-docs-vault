# Software Access Catalog

Owner: IT Security
Last reviewed: 2026-03-05

## Purpose

This catalog explains common access request routing. It is not a full entitlement matrix.

## Standard Access

- Email, Slack, calendar, and document storage are provisioned during onboarding.
- CRM access requires manager approval and Sales Operations confirmation.
- Production support tools require manager approval, system owner approval, and security training.
- Finance systems require Finance approval and role-based need.
- HR systems require People Operations approval and limited role scope.

## Request Process

Employees request access through the IT Support portal. Requests should include system name, business reason, requested role, duration if temporary, and manager approval.

## Temporary Access

Temporary access should include an expiration date. IT Security reviews open temporary grants monthly and may remove expired access without separate manager approval.

## Exceptions

Privileged access, API tokens, SSO changes, or access to customer data require IT Security review even when the manager approves the request.
